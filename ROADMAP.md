# Roadmap

Status legend: ✅ done · 🔜 in progress / next · ⬜ future.

## Phase 1: Personal Use

- ✅ Single entry point (`START-HERE.md`) and one-line session start.
- ✅ Deepened session lifecycle (`workflows/`).
- ✅ Customizable core identity and goals; isolated profiles.
- 🔜 Use the startup workflow daily across projects.
- 🔜 Track repeated mistakes in `core/mistakes.md`.

## Phase 2: Public-Quality Repo

- ✅ Skills layer (`skills/`) — reusable agent roles.
- ✅ Practical integrations (Claude Code, Codex, Cursor, ChatGPT).
- ✅ Expanded docs (concepts, context hierarchy, project memory, personalization).
- ✅ Neutral `profiles/example-developer/` for forkers.
- ⬜ Real case studies / examples from more stacks.
- ⬜ Diagrams.

## Phase 3: Automation

- ✅ `init-project.sh` — safe, idempotent template installer with bridge + git awareness.
- ⬜ `audit-project.sh` — check a project has valid memory files and a bridge.
- ⬜ `extract-lessons` — turn a session into a summary.
- ⬜ `update-agents` — refine `AGENTS.md` from project history.

## Phase 4: Skills and Agents (advanced)

- ⬜ Subagent orchestration patterns for parallel work.
- ⬜ MCP-compatible project memory.
- ⬜ Automated validation checklists.

## Phase 5: Community

- ✅ Contribution guidelines.
- ⬜ Profiles for students, researchers, founders, and teams.
- ⬜ Examples across different stacks.
