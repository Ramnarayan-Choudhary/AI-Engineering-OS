# Feature Development Playbook

## Purpose

Use this when building a new feature.

## Process

1. Define the feature in one sentence.
2. Define user value.
3. Define success criteria.
4. Inspect existing architecture.
5. Propose design.
6. Break into tasks.
7. Implement incrementally.
8. Test.
9. Review.
10. Document.

## Agent output

```markdown
## Feature summary
## User value
## Design
## Task breakdown
## Validation plan
## Risks
```
