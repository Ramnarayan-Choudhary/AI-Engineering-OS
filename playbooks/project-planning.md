# Project Planning Playbook

## Purpose

Use this when starting a new project.

## Process

1. Define mission.
2. Define users.
3. Define problem.
4. Define MVP.
5. Define non-goals.
6. Choose stack.
7. Design architecture.
8. Define milestones.
9. Define risks.
10. Define validation.
