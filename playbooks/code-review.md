# Code Review Playbook

## Process

1. Understand intent.
2. Read diff.
3. Check correctness.
4. Check edge cases.
5. Check architecture fit.
6. Check tests.
7. Check security and data handling.
8. Separate blocking issues from optional improvements.

## Output

```markdown
## Summary
## Blocking
## Important
## Optional
## Suggested tests
## Verdict
```
