# Mistakes and Lessons

## Purpose

This file prevents repeated mistakes.

Whenever an AI assistant, workflow, or project repeatedly fails in the same way, add a rule here.

## Standing rules

- Do not start coding before planning for large tasks.
- Do not rewrite working systems without approval.
- Do not add dependencies without explaining why.
- Do not rename public APIs casually.
- Do not skip validation.
- Do not claim tests were run if they were not.
- Do not expose secrets.
